# 资源引入

# 内部资源

aml 中的资源引入，是通过配置全局路径、文件目录和文件名的方式来实现的，demo 如下：

<!--
  path_config php             // 默认值为 normal，单行命令，重复配置将覆盖，可不配置 path_config
  path css css                // 配置 css 全局路径，单行命令，重复配置将覆盖，可不配置 path
  css lib example             // 配置引入的 css 文件，多次配置将追加引入
  css_min lib reset example   // 配置引入已压缩的 css 文件，多次配置将追加引入
  css index index
  js lib example              // 配置引入的 js 文件，多次配置将追加引入
  js_min lib example          // 配置引入已压缩的 js 文件，多次配置将追加引入
  js index index
-->

以上配置输出后将会变成：

<!DOCTYPE HTML>
<meta charset=utf-8>
<title></title>
<meta name=viewport content='width=device-width, initial-scale=1, user-scalable=0'>
<script>
  document.getElementsByTagName('html')[0].style.fontSize = innerWidth / 7.2 + 'px'
</script>
<link rel=stylesheet href=css/lib/example.css>
<link rel=stylesheet href=css/lib/reset.min.css>
<link rel=stylesheet href=css/lib/example.min.css>
<link rel=stylesheet href=css/index/index.css>

<script src=/static/js/lib/example.js></script>
<script src=/static/js/lib/example.min.js></script>
<script src=/static/js/index/index.js></script>

css 和 js 的资源引入逻辑其实是一样的，都是用 ${全局路径}/${目录路径}/${文件名}${文件后缀} 的格式将代码插入到模板当中，单行命令内的剩余参数会当做一个个的文件被追加进列表当中。

两个命令的 min 版本其实只是在写入资源的时候，在文件名后面添加 min 后缀。

path 与 path_config 命令都是用于配置全局路径的，区别在于 path 用于配置单个资源路径，而 path_config 用于导入预设配置，预设配置一共有三个，全局路径的值分别如下：

|- normal
|   |- img img
|   |- css css
|   |- js js
|
|- dotnet
|   |- img ../../Content/Images
|   |- css ../../Content/CSS
|   |- js ../../Scripts
|
|- php
|   |- img /static/img
|   |- css /static/css
|   |- js /static/js

# 外部资源

如果项目中需要引入外部资源，或资源路径独特，不便于配置的，可以使用 res 命令，写法如下：

<!--
  res css //a.b.c/d.css  // 配置引入的 css 文件，多次配置将追加引入
  res js //e.f.g/h.js?2  // 配置引入的 js 文件，多次配置将追加引入
-->

输出结果：

<!DOCTYPE HTML>
<meta charset=utf-8>
<title></title>
<meta name=viewport content='width=device-width, initial-scale=1, user-scalable=0'>
<script>
  document.getElementsByTagName('html')[0].style.fontSize = innerWidth / 7.2 + 'px'
</script>
<link rel=stylesheet href=//a.b.c/d.css>

<script src=//e.f.g/h.js?2></script>

TIP

资源引入的顺序与配置时的顺序将保持一致。

# 资源版本

为了解决浏览器资源缓存机制带来的烦恼，开发中会给引用的资源添加版本号，在 aml 中提供了 version 命令用于配置当前的资源版本号：

<!--
  js index index
  version 02281507
-->

输出结果：

<!DOCTYPE HTML>
<meta charset=utf-8>
<title></title>
<meta name=viewport content='width=device-width, initial-scale=1, user-scalable=0'>
<script>
  document.getElementsByTagName('html')[0].style.fontSize = innerWidth / 7.2 + 'px'
</script>

<script src=js/index/index.js?02281507></script>

WARNING

使用 res 命令引入的资源将不受版本号配置影响。

# 图片资源

开发离不开图片资源，为每一个图片标签写上一模一样的路径前缀，是很痛苦的事，虽然可以复制粘贴，但总归是不够灵动。为了解决这个问题，aml 提供了图片路径配置的解决方案，这也是 aml 诞生的目的。

<!---->

<img src=$logo$>
<img :src='"$" + index + "$"'>

输出为：

<!DOCTYPE HTML>
<meta charset=utf-8>
<title></title>
<meta name=viewport content='width=device-width, initial-scale=1, user-scalable=0'>
<script>
  document.getElementsByTagName('html')[0].style.fontSize = innerWidth / 7.2 + 'px'
</script>

<img src=img/logo.png>
<img v-bind:src='"img/" + index + ".png"'>

TIP

aml 会将 img 标签的第一个 $ 替换为配置好的图片全局路径，第二个 $ 替换为 .png，所以在开发中需要注意，不要在 img 标签上使用 $ 符号。

因为只检测 img 标签，所以对其他标签不存在效果。
