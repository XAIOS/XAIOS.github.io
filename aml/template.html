# 页面模板

aml 会读取 aml 文件的内容，将其整理后插入到预设模板，输出到目标格式中。提供两个预设模板，默认模板的内容如下：

<!DOCTYPE HTML>
<meta charset=utf-8>
<!-- title -->
<meta name=viewport content='width=device-width, initial-scale=1, user-scalable=0'>
<!-- meta -->
<script>
  document.getElementsByTagName('html')[0].style.fontSize = innerWidth / 7.2 + 'px'
// script
</script>

模板中的三个注释分别是页面标题，页面元数据，页面自定义脚本的插入接口，在 aml 文件中配置的相关内容将被处理后替换到指定地方，如果需要修改默认的页面架构，对模板页进行一点修改就可以了。

模板的语法其实就是 HTML 的语法，借助三个接口，很简单的就可以完成模板创建了。

有时候会遇到特别的项目，需要特别的模板来方便我们的开发，aml 中提供一个 template 命令，用于引入自定义的模板：

<!--
  template 1920 // 默认为 default，单行命令，重复配置将覆盖，可不配置 template
-->

这时 aml 会读取模板目录中的 1920.html 文件，将其作为模板使用，如果需要更多的自定义模板，只需要创建好之后将其移动到 aml 目录的 template 目录下，就可以在 template 命令中使用了，如果访问的文件不存在，将读取 default.html 作为模板。
