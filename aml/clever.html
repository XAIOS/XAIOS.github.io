# Clever Page

aml 诞生之时，Clever Page 的原型已经组建好了八天，可以说 aml 和 Clever Page 是共同诞生，一起成长的，所以 aml 中也为 Clever Page 的开发提供了很多便利。

使用 HTML 开发的话：

<div id=app v-cloak>
  <cp-page name=index>
    <cp-roll></cp-roll>
    <img v-bind:src=img touchable v-on:click.prevent=Click>
  </cp-page>
  <cp-wrap name=alert>
    <cp-pop alive></cp-pop>
  </cp-wrap>
</div>

如果使用 aml 进行开发：

<root>
  <page name=index>
    <roll></roll>
    <img :src=img @click=Click>
  </page>
  <wrap name=alert>
    <pop alive></pop>
  </wrap>
</root>

两者可以达到同样的效果，而 aml 会针对 root/page/roll/wrap/pop 五个标签进行转换，还会针对 img 标签进行点击事件的检测和补全，针对属性和事件绑定进行补全。于是我们可以写更少的，更贴合实际的代码，来实现同样的，甚至更多的事情。
