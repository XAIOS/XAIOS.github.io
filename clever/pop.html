# 弹窗层及弹窗对象

cp 的弹窗模块由 wrap 对象与 pop 对象共同组成，所以在文档中将两者一同介绍。

# 弹窗层/组

wrap 对象是 cp 中的一个弹窗组对象，通过实例中相关属性控制是否显示，对象默认带有 cp-wrap 的 class 名，可用于全局 wrap 对象样式修改，demo 如下：

<div id=app>
  <cp-wrap name=ralert></cp-wrap>
</div>

// index.js
new App

执行上述代码后，将在 cp 中创建一个 name 值为 ralert 的 wrap 对象，cp 中因为默认注册了 ralert 属性，所以不需要多作处理。现在在控制台中运行一句命令：

app.ralert = true

可以看到，页面上出现了一个半透明的黑色遮罩，这个就是 wrap 对象的弹窗层，但因为当前这只是一个无内容的对象，所以什么都没有。

# 弹窗对象

通过以下代码，添加一个 pop 对象到 wrap 对象中：

<div id=app>
  <cp-wrap name=ralert>
    <cp-pop name=rule></cp-pop>
  </cp-wrap>
</div>

// index.js
new App

在控制台中运行一句命令，将 pop 对象显示出来：

app.ralert = 'rule'

虽然此时的屏幕上依旧只能看到一个半透明的黑色遮罩，但其实名为 rule 的 pop 对象已经显示了出来，只是没有对其进行样式设置，所以无法看到。

pop 对象默认是居中定位的，带有格式为 ${wrap.name}_${pop.name} 的 class，也就是上面例子中的 pop 对象自带的 class 值为 ralert_rule。

TIP

在动效方面，pop 对象采用与 page 对象一样的切换机制，即在 pop 对象显示出来后，将拥有名为 show 的 class，可用于定制动画内容。

wrap 对象与 pop 对象的关系，和 cp 实例与 page 对象的关系差不多，都是通过值匹配的形式，去控制当前显示的内容，page 对象对应 cp 的 page 属性，而 pop 对象则对应 cp 上与 wrap 对象同名的属性。

另外，pop 对象也拥有 bottom 及 once 两个属性，作用与 page 对象的两个同名属性一致，但在 bottom 属性存在时弹窗大小将放大为全屏幕大小，开发时需要注意定位的相关问题。

TIP

wrap 对象可独立于 pop 对象存在，但 pop 对象必须为 wrap 对象的子对象，因为 pop 对象的显示是与父级的 wrap 对象绑定的，当 pop 对象不为 wrap 对象的子对象时，可能出现程序错误。

# 多个弹窗和多个弹窗层

通过多次创建 wrap 对象和 pop 对象，可以创建多个弹窗和弹窗层，通常情况下建议按视觉层级划分，即一级弹窗，将会覆盖在其上，与其同时存在的二级弹窗，如此类推的三级四级弹窗等，因为 cp 中的弹窗层将依据在 html 中的编写层级，越后写的层级越高。按弹窗类型分组可能导致 html 结构混乱，在数据结构中也不利于整理，按视觉层级划分的话，通常只有两个弹窗组，在处理上会非常便利。

<div id=app>
  <cp-wrap name=ralert>
    <cp-pop name=rule></cp-pop>
  </cp-wrap>
  <cp-wrap name=palert>
    <cp-pop name=info></cp-pop>
  </cp-wrap>
</div>

// index.js
new App({
  data: {
    palert: ''
  },
  mounted() {
    this.ralert = 'rule'
    this.palert = 'info'
  }
})

# 只有一个弹窗的弹窗组

虽然弹窗组内允许有多个弹窗的存在，但在一些场景中，我们只需要一个弹窗在内即可，如很常见的重写浏览器 alert 方法等，demo 如下：

<div id=app>
  <cp-wrap name=alert>
    <cp-pop alive>{{alert}}</cp-pop>
  </cp-wrap>
</div>

// index.js
new App({
  mounted() {
    this.alert = '提示内容'
  }
})

此处没为 pop 对象设置 name 属性，而是添加了 alive 属性到其上，代表这是一个无名的 pop 对象，随着 wrap 对象的显示而显示，始终存在。此时 pop 对象自带的 class 格式变为 pop_${wrap.name}，同时失去了通过 show 的增删来修改动画的能力，但可以借助 wrap 对象在显示时会存在的名为 cp-wrap-show 的 class 来实现效果。

由于 pop 对象的显隐与 wrap 对象同名属性值脱钩，就产生了更多的可能性，如上述例子中，值将作为文本输出到界面中，添加一些样式和按钮，就可以实现一个弹窗提示框了。

# 弹窗尺寸自适应

由于弹窗是脱离文档内容的，其大小通常不会与外部环境相影响，但如果一个弹窗在设计图中的尺寸太大，考虑到小屏幕设备的体验，还是需要对它的尺寸进行计算。

操作方式也是使用 suit 指令，但由于 pop 对象内部环境的原因，不能直接在 pop 对象上添加自定义指令，而应该这样写：

<div id=app>
  <cp-wrap name=ralert>
    <cp-pop name=rule suit=600,1000></cp-pop>
  </cp-wrap>
</div>

这样子尺寸就可以被传入到对象的内部了。

# 全屏弹窗

直接创建的 pop 对象，都是居中定位的，但有些时候我们需要的是随意定位，如提示分享弹窗，需要的是图片居右上定位，这个时候为 pop 弹窗添加一个属性，即可开始自由布局：

<div id=app>
  <cp-wrap name=ralert>
    <cp-pop name=share full></cp-pop>
  </cp-wrap>  
</div>

添加了 full 属性后，弹窗的尺寸将被设置为全屏幕大小，就好像 page 对象一样。

# 弹窗样式

直接在 pop 对象上添加的样式无法传递到内部的节点上，如果需要为弹窗添加自定义样式，需要使用属性传递，通常情况下不会遇到这种情况：

<div id=app>
  <cp-wrap name=ralert>
    <cp-pop name=rule child_class=test></cp-pop>
  </cp-wrap>
</div>
