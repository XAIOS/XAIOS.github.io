# 闲谈

Clever Page （下称 cp）的雏形是 2018.05.17 那天，从过去项目经验中抽象出来的一个框架，那时候懒得命名，直接叫 XPage。最初那版的 js 源码只有 100 行，内容也仅包括页面、滚动区域、弹窗层以及弹窗对象，而且其中存在的 bug 太多，可扩展性也不强，但用起来就很舒服，自那以后我的项目少写了超多代码。人一旦懒下来就很难停止，就像滚雪球一样，在这个大大大大大的惯性下，在同月的 25 号，我编写出了 aml 的雏形，花了四五个钟头，用 93 行代码，省下了日后项目中平均上千的字节数，当然那不在本文范围内了。

cp 在同年七月三十号正式更名并立项，发布了 1.0.0 版本，到写下这份文档的时候，版本号已经是 1.8.0 了，js 源码也增加到了 652 行，还有 208 行的 stylus 源码，但更新日志是不可能有的，因为太懒了。

第一份文档在同年八月十八日编写完毕，其中包括的内容比起最初的雏形，已经是翻了几倍，但文档内容是很简短的，采用的格式也是目录式的，完全是面向开发人员的版本，后面想了想觉得应该有份正式的完整的文档，加上也比较闲，所以创建了这份文档，也算是整理整理自己的开发经验。

时间节点记得那么清，主要是因为，发了朋友圈。

# 概念

cp 是基于 Vue 的一个框架，开发目的是更进一步地规范页面的结构，做到即使不使用单文件组件，也可以高度地模块化组件化开发，为开发者提供高扩展性高可维护性的高效开发环境。

前端的页面，从视觉上主要可以分为页面、弹窗、滚动区域等模块，这些模块本质上都是一个容器。cp 的作用就是在页面上规划好大容器，让开发者将内容填充到这些容器中，并在 js 中通过简单的逻辑调用，将对应的容器显示出来，容器之间的关系有互斥也有相互独立，当然也可以嵌套。

cp 的实例，可以理解为当前这个站点的一个抽象，通过对实例的页面属性和弹窗属性等进行描述，显示对应的页面和弹窗，完全的面向对象开发，而将机械化的操作交给程序。

# 注意事项

cp 是站在个人角度进行开发的，没有团队和成熟的经理进行测试和需求明确，成长主要是靠开发项目，项目出问题了就修复，常见的需求就封装，所以也算是通过了实际测试，但一人之力，难免会有疏漏，况且还是一懒人之力，所以开发的时候有几点需要注意：

  • cp 未对 v-cloak 属性进行处理，请自行在 CSS Reset 中编写相关逻辑。
  • cp 是基于微信浏览器进行开发及测试的，用于其他浏览器时或需按实际情况进行相应修改。
  • cp 不影响 Vue 实例的创建，但单纯的 Vue 实例无法使用 cp 提供的部分功能。

WARNING

cp 在微信浏览器和 iOS 浏览器上运行时将阻止大部分情况下的默认滚动事件，影响滚动操作，可通过使用滚动组件的形式解决，如果在开发中遇到滚动上的问题，可移除相关的 cp 代码使滚动机制恢复为原来的样子。

WARNING

本文档编写时 Vue 的版本是 2.5.22，其后更新中带来的新旧语法变更，可能影响到 cp 的核心部分，以及文档的准确性，开发时需要注意，文档与 cp 将尽可能及时更新。
