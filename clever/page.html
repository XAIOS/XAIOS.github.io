# 页面对象

cp 提供的页面容器对象，通过实例的 page 属性控制显示与否，对象默认带有 cp-page 的 class 名，可用于全局 page 对象样式修改。

# 对象的创建与显示

page 对象是 cp 最基础也是最常用的组件，是项目中一个页面的抽象，一个最简单的 page 对象可以通过以下代码创建：

<div id=app>
  <cp-page name=index>
    <!-- 页面内容 -->
  </cp-page>
</div>

/* index.css */
/* page 对象默认带有格式为 page_${name} 的 class 名，无需额外添加 class */
.page_index
  background #272822

// index.js
new App

TIP

渲染后的 DOM 结构可在控制台中查看，此后不多作描述。

在浏览器中运行上述代码之后，可以在页面上得到一片空白，然后在控制台中再执行一句命令：

app.page = 'index'

可以看到，页面上出现了背景色为 #272822 的一个页面，这时我们所创建的 page 对象才被显示出来。

WARNING

page 属性是 cp 实例的私有属性，为了方便使用，将其表现为了公开属性，但在开发中应注意 page 属性的赋值，以免页面显示异常。

page 属性代指的是当前需要显示的页面的名字，在创建 page 对象时会为其提供一个 name 属性，当 name 属性与 cp 实例的 page 值一致时，页面将会被显示，所以要注意，page 对象的名字应是唯一的。

以下代码也是合法的：

<div id=app>
  <cp-page name=0></cp-page>
</div>

// index.js
new App({
  mounted() {
    this.page = 0
  }
})

注册的 page 对象将会如期显示出来，因为在值判断上使用的是 ==。另外页面显隐操作建议在 cp 实例完成 DOM 挂载后执行，以免出现显示异常。

# 可滚动页面的创建

实际开发中不可避免会遇到可滚动的页面，这个时候只需要为页面对象添加一个属性，就可以实现可滚动页面：

<div id=app>
  <cp-page name=index roll></cp-page>
</div>

添加 roll 属性，将把 roll 对象和 page 对象结合起来，实现滚动页面的效果，roll 对象将在后文进行讲解，目前只需要知道添加这个属性，在页面内容超出页面高度的时候，可以滚动，就行了。

TIP

页面可滚动之后，很常见的一个需求是在其上添加一个依据浏览器窗口定位的区域，虽然使用 position: fixed 可以很快解决这个问题，但在 iOS 上滚动时可能会出现元素闪动之类的问题，cp 中建议通过固定区域插槽来实现这个效果：

<div id=app>
  <cp-page name=index roll>
    <img slot=fixed>
    <template slot=fixed>
      <img>
      <img>
    </template>
  </cp-page>
</div>

fixed 插槽内的元素将处于单屏环境中，使用单屏页面的元素处理方案即可完成相应的布局处理。

# 页面的动效

默认提供的页面切换效果是淡入淡出，如果定制的话，可以参考下面的例子：

<div id=app>
  <cp-page name=index></cp-page>
  <cp-page name=game></cp-page>
</div>

/* index.css */
.page_index
  transform translateY(-100%)
  transition all .6s

.page_game
  transform translateY(100%)
  transition all .6s

.show.page_index, .show.page_game
  transform none

执行一次两个页面间的显隐切换，可以看到淡入淡出的同时，有上下出入的切换效果。

TIP

显示中的 page 对象带有名为 show 的 class，通过 css 选择器设置状态，利用 transition 进行补间，可以非常简单地添加动画效果，通过下面的例子，还可以设置页面内元素的动效：

<div id=app>
  <cp-page name=index>
    <img>
    <img class=test>
  </cp-page>
</div>

/* index.css */
.page_index img
  opacity 0
  transition-duration .6s
  transition-property all
  transition-timing-function linear

.show.page_index img
  opacity 1

.test
  transition-delay .6s

TIP

很多时候我们希望一个页面内的动画只执行一次，以减少多次切换时的用户体验时间，这个时候我们可以在 page 对象上添加一个属性解决：

<div id=app>
  <cp-page name=index once></cp-page>
</div>

/* index.css */
.showed.page_index img
  transition-delay 0s

代码内容承接上文，核心的部分如上，添加了 once 属性的 page 对象会在显示一次后被添加名为 showed 的 class，在此选择器下可使元素失去动画，推荐仅设置动画时延为 0s 即可。

# 使页面对象置底显示

当输入框被聚焦时，移动端的浏览器会弹出软键盘，页面的高度会因此被挤压，普通页面会滚动到使输入框可视的位置，但因为 cp 将整个容器的高度预设为了屏幕的高度，所以内容不会因为可视区域的高度坍塌而变化，简单来说，就是如果页面的下方存在一个输入框，当期被聚焦的时候，软键盘会弹出，将页面的下半部分，包括输入框一起给盖住。

在 cp 中为 page 对象提供了一个 bottom 属性，声明了该属性的页面，将会区别于其他页面的置顶定位，改为相对屏幕置底定位，在不存在软键盘的情况下看不出区别，但如果弹出了软键盘，就可以发现，页面整体像是被软键盘顶起了一样向上移动，下方存在的输入框也就进入了视野当中：

<div id=app>
  <cp-page name=index bottom></cp-page>
</div>

在页面上方存在输入框时不需要额外操作，下方存在输入框的时候可以通过 bottom 属性解决，如果是页面的中间存在输入框或是整体布满输入框的话，需要的改动会更多，比如设置一个滚动区域，在聚焦事件中将控件滚动到可视区域当中等等，因为出现的频率较低，所以也不多作说明了。
