# 基础页面架构

因为本人接触的项目规模都不大，所以除了最开始学的时候用过构建工具外，就几乎没接触了。cp 诞生自小项目，所以直到写这段文字之前，我都没想过将 cp 放在构建环境中，要怎么使用。

思考了一两分钟之后，结合从业以来的项目经验和身边同事的车祸经历，最后决定不去考虑这个问题了，毕竟要用构建的，项目规模一般不低，而 cp 的格局目前是很小的，个人还是推荐使用在小项目当中，当然概念是通用的，理解了 cp 的设计理念，也可以应用在大型的项目当中。

从普通页面开发的角度出发，下面先给出一个最最基础的 demo，文档中所有的示例都将基于这个架构：

<!DOCTYPE HTML>
<meta charset=utf-8>
<meta name=viewport content='width=device-width, initial-scale=1, user-scalable=0'>
<script>
  // cp 的样式是基于 rem 单位设置的，且默认设计图宽度为 720
  // cp 的默认样式涉及到 rem 的只有 Toast 及加载提示模块
  document.getElementsByTagName('html')[0].style.fontSize = innerWidth / 7.2 + 'px'
</script>
<link rel=stylesheet href=clever.min.css>
<link rel=stylesheet href=index.css>

<div id=app></div>

<script src=vue.min.js></script>
<script src=clever.min.js></script>
<script src=index.js></script>

// index.js
// App 继承自 Vue，内部已设定了挂载节点 id 为 app，可以在构建 cp 实例时再次声明以更新设置
// Vue 不建议在继承中设置挂载节点，所以在开发环境下会有错误提示信息，其实并没有影响
new App

TIP

上述代码运行之后，页面的实例将被挂载到 window 对象的 app 属性当中，在全局环境下可以通过 app 访问当前 cp 实例，以便在控制台中进行调试。
