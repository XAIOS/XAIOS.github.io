# 对实例内部环境的影响

为了实现功能的封装，cp 内部注册了很多的属性和方法，这些属性和方法都是可以被修改的，但通常情况下不建议修改，因为会影响到内部的相关操作，除非该部分的操作在项目的生命周期内不会被调用。

是的，这些半私有的属性选用了不少常见的变量名，可能会对开发者造成困扰，但本人并不在意。

# 实例内的属性

  • bgm

表示背景音乐的播放状态，在 data 内注册，默认值为 0，详见背景音乐模块。

  • load

表示当前已加载数量的计数器，在 data 内注册，默认值为 0，与资源预加载部分有关，如不使用 cp 推荐的预加载方式，可随意使用 load 属性。

  • page

表示当前显示的页面名字，在 data 内注册，默认值为空字符串，详见页面模块。

  • alert | ralert

用于显示 name 为 alert 或 ralert 的 wrap 对象，在 data 内注册，默认值为空字符串，因为我经常使用所以一并封装到 cp 当中，纯属个人习惯，与内外逻辑并无严密关联，详见弹窗模块。

  • _isIOS | _isWeChat

通过 ua 判断，当前是否 iOS 环境与微信环境，

  • _orientation_msg

横屏模式下的提示文案，默认值为 '请使用竖屏模式浏览'，需在 mounted 生命周期前设置，详见横屏模式处理部分。

  • _orientation_cancel

是否取消横屏的处理，值为真时将不在屏幕旋转的时候进行额外处理，需在 mounted 生命周期前设置，详见横屏模式处理部分。

# 实例内的对象

  • bgm_music

背景音乐对象，详见背景音乐模块。

  • _query

当前 url 中查询参数反序列化后的对象，非响应式，只包含初始化实例时 url 中的查询参数。

  • _orientation

横屏模式遮罩的 Vue 实例，在触发过一次横屏之后会被创建，一般情况下不需要理会，详见横屏模式处理部分。

# 实例内的方法

  • AlertClose | RalertClose

用于关闭 name 为 alert 或 ralert 的 wrap 对象，可被重写覆盖。

  • InitBgm

初始化背景音乐，详见背景音乐模块。

  • Bgm

背景音乐操作方法，用于暂停或开始播放，详见背景音乐模块。
