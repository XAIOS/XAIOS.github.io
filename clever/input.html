# 输入框对象

在微信某一个版本的更新之后出现了一个 bug：当从输入框中移除焦点的时候，body 的滚动高度会发生变化，但视图并未移动，导致视图和实际位置有偏差，简单来说就是视图获取焦点异常，无法正常触发点击之类的事件。

官修是不可能的，即使修复了，也不可能保证每个人的微信都更新了，最后还是得自己去修复。

其实修复的逻辑很简单，就是在输入框的失焦事件中，将 body 的滚动高度设置为 0，考虑到开发效率，cp 中将输入框封装成了一个对象，内置了对那个 bug 的修复。

demo 如下：

<div id=app>
  <!-- 渲染为 <input type=text v-model=name> -->
  <cp-input v-model=name></cp-input>

  <!-- 渲染为 <input type=tel> -->
  <cp-input type=tel></cp-input>

  <!-- 渲染为 textarea -->
  <cp-input textarea></cp-input>
</div>

使用方式上与 input 标签或 textarea 标签一样，建议只用 input 对象代替输入框类控件。
