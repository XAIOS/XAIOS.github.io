# 选择器对象

# 基础选择器

开发中遇到添加选择器的需求时，很自然地会使用 select 标签，但这个标签存在不少的限制，如样式和事件等等，所以很多框架（主要是 UI 框架）都封装有自己的选择器，cp 也不例外，主要是为了统一 Android 和 iOS 两端的表现。

由于样式与设计稿关联十分密切，所以 cp 只封装了最核心的逻辑框架，具体样式需要按设计进行开发，demo 如下：

<div id=app>
  <!-- 可使用 v-model 进行属性的双向绑定，实例中的同名属性值将为选择器的值，
       可对其进行赋值更新选择器显示内容，但对其赋不存在于 list 中的值时操作将被无效化 -->

  <!-- placeholder 属性可用于设置占位文本，默认为'选择' -->

  <!-- 可用 lazy 属性表示 list 的传值可信任，存在该属性时将跳过内容格式化，该值无需动态 -->

  <!-- 可用 disabled 属性禁用 select 对象，支持隐式类型转换比较，为 true 值时组件将被禁用 -->

  <!-- 可监听 select 事件用于捕获点击的选项，接收一个参数，代表当前点击项的值 -->

  <cp-select name=area v-model=area placeholder='例子' :list=list @select=Sel></cp-select>
</div>

/* index.css */
/* cp 源码中不提供这部分样式，为了效果展示，提供简单的样式 demo */
/* select 对象带有格式为 `select_${name}` 的 class，设置 name 属性可以修改，可用于建立命名空间 */
/* 考虑到选择器在同一项目中很少存在多种样式的情况，编写 css 时也可以忽略命名空间 */
/* 如不考虑命名空间的问题，name 属性可以忽略，此时 select 对象将仅带有 cp-select-box 的 class */
.select_area
  /* 修改此类以提供 border、background 等外围容器样式，默认宽高 100% */
  &.cp-select-box
    width 60px
    height 26px
    outline 1px solid #000
    background #fff

  /* 修改此伪类以提供下拉提示小三角的样式，默认存在值为空字符串的 content 属性 */
  &.cp-select-delta::after
    margin-left 6px
    border-top 6px solid #000
    border-left 4px solid transparent
    border-right 4px solid transparent

  /* 修改此类以提供选项组样式 */
  .cp-select-group
    outline 1px solid #000
    background #fff

  /* 修改此类以提供选择项样式 */
  .cp-select-item
    height 26px

// index.js
new App({
  data: {
    // list 的修改将同步到 select 对象中，但值要求始终是数组，否则会报错
    list: [],
    area: ''
  },
  created() {
    /*
      list 数组的格式应为 [{key: '显示文本', val: '选项值'}]，允许存在其他属性
      当选择项不为对象值时，将选用该项的内容作为显示文本以及选项值，此时的 list 对象不可信任
      注意当 lazy 属性添加后将始终存在，list 对象的可信度应在开发中进行考虑
    */
    this.list = [{
      key: '澳门',
      val: 'value'
    }, '香港']
  }
})

TIP

创建 select 组件后将在 window 对象上注册名为 __cp_select_group 的数组，用于存放所有的 select 对象，开发中基本上不需用到。

TIP

select 对象一般用于简单的选择，复杂度较高的情况下建议使用其他 UI 框架或插件实现选择器。

# 手机地区选择

基于 select 对象二次封装的组件，更方便地选择手机区号，demo 如下：

<div id=app>
  <!-- 使用上基本与 select 对象一样，但移除了 list 和 lazy 属性 -->

  <!-- 默认可选 ['澳门','香港','大陆','其他']，添加 tw 属性后将追加可选 '台湾' -->
  <!-- 选项顺序不提供更改，需在源码中处理，或修改编译后的代码 -->

  <!-- 默认显示为正体中文，添加 simple 属性可切换为简体中文，但不会作用于设置的 placeholder -->

  <cp-phone-code v-model=area></cp-phone-code>
</div>

// index.js
new App({
  data: {
    // 根据选项返回对应区域的手机区号，不含 '+'，如选择大陆，值为 '86'
    area: ''
  }
})
