# 资源预加载

游戏活动类的项目一般涉及大量的图片资源，这部分的资源加载很多时候是很耗时的，所以一般情况下会有资源加载页存在，而 cp 也提供了一套资源加载方案用作参考，并内置了相关的属性，以下是 demo：

// index.js
// 所需加载的资源总数，包括图片量和接口请求量
PRE = 3
// 资源加载的路径前缀，可通过修改这个值来快速切换线上和线下的资源库指向
PATH = 'img/'
// 图片名列表
LOAD = ['title', 'logo']

new App({
  computed: {
    // 如果需要在加载页上同步加载进度的话，建议在计算属性中返回
    progress() {
      // 返回百分比
      return (this.load / PRE).toFixed(0) + '%'
    }
  },
  watch: {
    // cp 内置的属性，注册在 data 中，默认为 0，用作预加载的计数器
    load(val) {
      // 当计数器累计到总数时触发站点的数据初始化操作
      val == PRE && this.Init()
    }
  },
  methods: {
    Init() {
      // 数据初始化操作，如将页面切换到首页等等
      this.page = 'index'
    }
  },
  created() {
    // 在实例的生命周期内开始资源的预加载
    // 预加载全部图片
    LOAD.forEach(i => {
      var img = new Image
      // 图片加载完成后给计数器加一
      img.onload = () => ++this.load
      // 通过字符串拼接返回图片路径，可节省巨额代码量，但要求资源有统一性
      // 通常分类按图片后缀划分，如 png 和 jpg
      img.src = `${PATH}${i}.png`
      // 此处建议在 onerror 回调中为用户创建提示
    })

    // 举个例子的接口请求
    Post('api', {}, data => {
      // 在接口回调中为计数器加一
      this.load++
    })
  },
  mounted() {
    // 显示加载页
    this.page = 'loading'
  }
})

TIP

资源预加载的要点在于算出所需加载的资源数，在加载完成的回调中为计数器加一，计数器与总数的比值就是加载的进度，统计范围包括但不限于需要加载的图片和必要的接口请求。
