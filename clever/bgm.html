# 背景音乐

活动的背景音乐是稍微麻烦且需求很普遍的功能点，在 cp 中封装了几个属性和方法，用于快速创建并控制背景音乐，以下是 demo：

<!-- 音频在规范中需要用户至少产生一次交互才可以自动播放 -->
<div id=app @click.once=Bgm(1)>
  <!-- 音频开关图标 -->
  <img :src='"img/music_btn_" + bgm + ".png"' touchable @click.prevent.stop=Bgm>
  <!-- img/music_btn_0.png -->
  <!-- img/music_btn_1.png -->
</div>

// index.js
new App({
  created() {
    // 传入音频文件路径，初始化背景音乐
    // 方法内部检测到微信 jssdk 存在时会在 wx.ready 中调用播放
    this.InitBgm('music/bgm.mp3')
  }
})

下面是一些内置属性及方法的详解：

  • bgm

表示背景音乐的播放状态，在 data 内注册，默认值为 0，音乐开关状态下值分别为 1 和 0，不建议手动对其进行设置，因为这并不会影响到背景音乐对象，但可以对其进行除设置外的操作，如监听和计算等。

  • bgm_music

背景音乐的 Audio 对象，通常情况下不建议直接操作，调节音量等影响较小的操作除外。

  • Bgm

背景音乐控制方法，用于暂停或开始播放，直接对音频对象播放状态取反。

  • InitBgm

背景音乐初始化方法，传入音频文件路径，环境允许的情况下会自动播放。

TIP

cp 内封装的背景音乐模块功能点较少，如果需要更加复杂的操作，建议自行实现相关的逻辑。
